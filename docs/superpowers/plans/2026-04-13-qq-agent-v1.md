# QQ Agent V1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first working personal QQ bot stack with a Dockerized Claude CLI runner, strict container CPU and memory controls, and Rust-enforced command timeout.

**Architecture:** `NapCat` and `AstrBot` stay as integration layers, while a Rust `agent-runner` owns Docker lifecycle, HTTP execution requests, timeout enforcement, and normalized output. The Claude CLI runs in a dedicated Ubuntu 24.04 container that only mounts `/workspace` and `/state`, while upstream credentials stay outside the container behind a host-local API proxy.

**Tech Stack:** Rust (`axum`, `tokio`, `serde`, `bollard`, `tracing`), Docker Compose, Ubuntu 24.04, shell scripts, AstrBot plugin glue

---

## File Structure

- Create: `compose/docker-compose.yml`
  Defines the runtime topology, especially the `claude-runner` service with CPU, memory, PID, mount, tmpfs, and root filesystem controls.
- Create: `docker/claude-runner/Dockerfile`
  Builds the Ubuntu 24.04 image with Node.js, npm, Claude CLI, and a small entrypoint.
- Create: `docker/claude-runner/entrypoint.sh`
  Keeps the container alive in a predictable way and prepares writable directories.
- Create: `agent-runner/Cargo.toml`
  Rust package manifest and dependency declarations.
- Create: `agent-runner/src/main.rs`
  Binary entrypoint, tracing setup, configuration loading, and HTTP server boot.
- Create: `agent-runner/src/config.rs`
  Parses environment-driven configuration with validated timeout and Docker defaults.
- Create: `agent-runner/src/models.rs`
  Platform-neutral request and response types for API and internal execution results.
- Create: `agent-runner/src/docker_client.rs`
  Docker helpers for image, container lookup, container creation, start, inspect, and exec setup.
- Create: `agent-runner/src/timeout.rs`
  Timeout helper and kill escalation logic.
- Create: `agent-runner/src/exec.rs`
  Claude exec orchestration, output collection, and exit status normalization.
- Create: `agent-runner/src/server.rs`
  Axum routes for `/healthz` and `/v1/runs`.
- Create: `agent-runner/tests/config_tests.rs`
  Validates config defaults and timeout bounds.
- Create: `agent-runner/tests/http_api_tests.rs`
  Verifies HTTP request validation and timeout boundary behavior without requiring live Docker.
- Create: `scripts/bootstrap.sh`
  Creates local directories and prints next-step instructions.
- Create: `scripts/healthcheck.sh`
  Smoke-check script for the Rust service endpoints.
- Create: `astrbot/plugins/claude_runner_bridge/README.md`
  Defines the expected contract for the AstrBot-side plugin.

## Interface Contract

### agent-runner HTTP API

`POST /v1/runs`

Request body:

```json
{
  "platform": "qq",
  "conversation_id": "private:123",
  "session_id": "qq-user-123",
  "user_id": "123",
  "chat_type": "private",
  "cwd": "/workspace",
  "prompt": "hello",
  "timeout_secs": 120
}
```

The core request model must remain platform-neutral even though v1 only targets QQ.

- [ ] **Step 3: Write the minimal compose file**

Create `compose/docker-compose.yml` with:

```yaml
services:
  claude-runner:
    container_name: claude-runner
    build:
      context: ..
      dockerfile: docker/claude-runner/Dockerfile
    restart: unless-stopped
    stdin_open: false
    tty: false
    cpus: "2.0"
    mem_limit: 4g
    memswap_limit: 4g
    pids_limit: 256
    read_only: true
    tmpfs:
      - /tmp:size=256m,mode=1777
    environment:
      ANTHROPIC_BASE_URL: ${ANTHROPIC_BASE_URL:-http://host.docker.internal:9000}
      CLAUDE_CONFIG_DIR: /state/claude
    volumes:
      - ${AGENT_WORKSPACE_DIR:-/srv/agent-workdir}:/workspace
      - ${AGENT_STATE_DIR:-/srv/agent-state}:/state
    working_dir: /workspace
    extra_hosts:
      - "host.docker.internal:host-gateway"
    command: ["/usr/local/bin/entrypoint.sh"]
```

- [ ] **Step 4: Write the minimal Dockerfile**

Create `docker/claude-runner/Dockerfile` with:

```dockerfile
FROM ubuntu:24.04

ENV DEBIAN_FRONTEND=noninteractive

RUN apt-get update \
    && apt-get install -y --no-install-recommends \
       ca-certificates \
       curl \
       git \
       gnupg \
       bash \
       tini \
    && rm -rf /var/lib/apt/lists/*

RUN mkdir -p /etc/apt/keyrings \
    && curl -fsSL https://deb.nodesource.com/gpgkey/nodesource-repo.gpg.key \
       | gpg --dearmor -o /etc/apt/keyrings/nodesource.gpg \
    && echo "deb [signed-by=/etc/apt/keyrings/nodesource.gpg] https://deb.nodesource.com/node_22.x nodistro main" \
       > /etc/apt/sources.list.d/nodesource.list \
    && apt-get update \
    && apt-get install -y --no-install-recommends nodejs \
    && npm install -g @anthropic-ai/claude-code \
    && rm -rf /var/lib/apt/lists/*

COPY docker/claude-runner/entrypoint.sh /usr/local/bin/entrypoint.sh
RUN chmod +x /usr/local/bin/entrypoint.sh

WORKDIR /workspace
ENTRYPOINT ["/usr/bin/tini", "--"]
CMD ["/usr/local/bin/entrypoint.sh"]
```
